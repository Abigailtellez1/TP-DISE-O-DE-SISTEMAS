# Frontend Implementation Plan

This React + Vite frontend should mirror the backend we built in Spring Boot. The goal is to deliver features incrementally, keeping each module self-contained so the team can iterate in small chunks. All endpoints live at `http://localhost:8080`.

## Tech Stack / Architecture

- React 19 with functional components + hooks.
- React Router for navigation between feature areas.
- State management: component-level state + React Query (or simple fetch hooks) for data fetching/caching.
- UI Library: start with plain CSS (Tailwind or MUI can be added later if desired).
- API layer: `src/api/` with typed functions calling backend endpoints.

## Feature Modules / Deliverables

1. **User Profile Module**
   - Screens: View profile, edit profile form.
   - Endpoints:
     - `PUT /api/users/{userId}` to create/update a profile.
     - `GET /api/users/{userId}` to load data.
   - Flow:
     - Start with a text input to enter a `userId` (mocked SSO ID).
     - Form fields: email, name, preferred bedrooms.
     - Show success/failure feedback.

2. **Listing Management Module**
   - Screens: Listings table with pagination controls, create listing form, edit listing (patch) modal.
   - Endpoints:
     - `GET /api/listings?page={page}&size={size}`
     - `GET /api/listings/{id}`
     - `POST /api/listings`
     - `PATCH /api/listings/{id}`
     - `DELETE /api/listings/{id}`
   - Flow:
     - Show paginated table.
     - Provide filters for page/size.
     - Add “New Listing” button → form.
     - Row actions for edit/delete.

3. **Reviews Module**
   - Screens: Per-listing review list, create review form.
   - Endpoints:
     - `GET /api/listings/{listingId}/reviews?page={page}&size={size}`
     - `POST /api/listings/{listingId}/reviews`
   - Flow:
     - For a selected listing, fetch reviews and show in a list with rating + comment.
     - Form requires rating, comment, author ID (existing user).

4. **Notifications Module**
   - Screen: Notifications dashboard showing events per user.
   - Endpoints:
     - `GET /api/users/{userId}/notifications`
   - Flow:
     - Provide a user selector (reuse from profile module).
     - Show the list of notifications (listing title/bedrooms/timestamp).

## Iteration Order

1. **User Profile Module** – unlocks other features by allowing us to create users and set bedroom preferences.
2. **Listing Management Module** – build core CRUD with pagination.
3. **Reviews Module** – attach reviews to listings once listings/users exist.
4. **Notifications Module** – display notifications generated by the observer when listings match user preferences.

## Testing & Tooling

- Use Vite dev server for local development.
- Mock data with MSW or a simple fake `userId` until real SSO is available.
- Add basic unit tests for API hooks and components as time allows.
